# Leafiq

> 특별한 날을 기록하세요, 내 인생을 한 편의 산문집처럼 만들어드립니다.
https://leafiq.site

## 프로젝트 소개

**Leafiq**는 매일 쓰지 않아도 괜찮은 추억 기록 서비스입니다.

행복했던 순간, 슬펐던 기억, 잊고 싶지 않은 특별한 날들을
길어도, 짧아도 상관없이 산문처럼 남기고
시간이 지난 후 다시 돌아와 조용히 읽어볼 수 있는 나만의 공간입니다.

### 왜 Leafiq인가?

기존의 일기/기록 서비스는 "매일 써야 한다", "꾸준히 해야 의미가 있다"는 전제를 강요합니다.
하지만 많은 사람들은 매일 쓰는 건 부담스럽고, 그래도 가끔씩 찾아오는 특별한 날만은 어딘가에 남기고 싶어합니다.

Leafiq는 이 지점에 집중합니다.

- **부담 없는 기록**: "매일 써야 한다"는 압박 없이, 특별한 날에만 기록해도 충분합니다
- **시간이 만드는 가치**: 시간이 지나면 하나의 인생 산문집처럼 쌓이는 경험
- **선택적 공유**: 나만 보기 또는 "모두의 산문집"에 공유하여 다른 이들과 감정을 나눌 수 있습니다

### 핵심 가치

- 마음이 움직였을 때만 쓰는 기록
- "기록 실패"에 대한 죄책감이 없는 서비스
- 과거의 나를 다시 만나는 감동적인 경험

## 기술 스택

- **Runtime**: Node.js
- **Web Server**: Express.js
- **Frontend**: Vanilla JavaScript
- **API Communication**: Fetch API
- **Containerization**: Docker

## 주요 기능

### 필수 기능 (MVP)

- **회원 인증**: 이메일 기반 회원가입 및 로그인
- **추억 기록 CRUD**:
  - 제목, 내용, 날짜, 공유 여부를 포함한 기록 작성
  - 작성한 기록 수정 및 삭제
  - 시간 역순으로 정렬된 기록 목록 조회
- **나의 산문집**: 내가 작성한 모든 기록을 카드형 UI로 확인
- **모두의 산문집**: 공유가 활성화된 다른 사용자들의 기록 열람
- **선택적 공유**: 각 기록마다 공개/비공개 설정 가능 (개발 예정)
- **프로필 관리**: 닉네임 및 비밀번호 변경

### 향후 확장 기능

- 감정 태그 (행복/슬픔/평온 등)
- 캘린더 뷰
- 기록 통계 (연도별/감정별)
- 공유하기
- 소셜 로그인

├── Dockerfile             # Docker 이미지 빌드 설정

## 시작하기

### 사전 요구사항

- Node.js 18 이상
- npm

### 로컬 개발 환경 설정

1. 저장소 클론

```bash
git clone <repository-url>
cd kakao-community-fe
```

2. 의존성 설치

```bash
npm install
```

3. 서버 실행

```bash
node app.js
```

서버가 `http://localhost:3000`에서 실행됩니다.

### Docker를 사용한 실행

1. Docker 이미지 빌드

```bash
docker build -t kakao-community-fe .
```

2. 컨테이너 실행

```bash
docker run -p 3000:3000 kakao-community-fe
```

### 주요 라우트

| 경로 | 페이지 | 설명 |
|------|--------|------|
| `/` | 홈 (로그인) | 메인 진입점 |

| `/posts/edit` | 기록 수정 | 기존 추억 수정 |
| `/profile` | 프로필 수정 | 닉네임 및 사용자 정보 관리 |
